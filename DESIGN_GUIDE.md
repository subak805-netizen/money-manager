# 머니 — 디자인 가이드

> UI/디자인 작업 전 이 문서를 먼저 읽고 따른다. 디자인이 바뀌면 이 문서를 무조건 업데이트한다.

## 파운데이션 토큰 시스템 (FPF와 통일)
- **FPF 앱(`fpf-manager`)의 UI 파운데이션과 동일한 토큰 구조·이름 규칙**을 쓴다. 색 값만 머니의 깔끔한 뉴트럴.
- **마스터 기준표:** `mockup-foundation-master.html` (색·타이포·간격·형태·컴포넌트 한눈에). 새 컴포넌트는 여기 + 이 문서 기준.
- **원시 토큰(테마 무관, FPF와 공통):** 간격 `--sp-1~10`(base 4) / 컨트롤높이 `--h-s/m/l/touch`(28/34/42/44) / 타이포 `--t-caption/label/body/title/h`(11/12/13/15/18) + `--t-display`(30, 금액) / 폰트굵기 `--fw-reg/med/bold/x`(400/600/700/800) / 반경 `--r-sm/md/lg/pill`(10/14/20/999) / 그림자 `--el-1/el-2` / `--content-max`(머니=520, FPF=760).
- **의미 토큰(FPF 이름 규칙):** `--page-bg --card-bg --card-bd --text --t2 --t3 --input-bg --input-bd --accent --accent-bg --on-accent --ok --red --warn --chip --chip-ic --dark`. 색·선·반경 하드코딩 금지, 토큰만 사용.
- 앱 `_live.html`의 `:root`는 이 토큰을 정본으로 정의하고, 본문 CSS가 쓰던 옛 짧은 이름(`--bg --line --income --neg --r --sh` 등)은 **값이 같은 별칭**으로만 연결돼 있다(점진 마이그레이션). 새 코드는 별칭 말고 파운데이션 이름을 쓴다.
- 테마 전환(data-theme)은 아직 미적용 — 머니는 뉴트럴 1테마. 나중에 레트로 등 추가 시 FPF처럼 토큰만 갈아끼운다.

## 반응형 레이아웃 (모바일 앱 ⇄ PC 대시보드)
- **< 960px (폰):** 지금의 모바일 앱 — 단일 열, 하단 탭바(가계부/예산/자산/예측) + FAB(＋). `.shell` 520px 중앙.
- **≥ 960px (PC):** **왼쪽 고정 사이드바**(264px: 로고+메뉴+프로필) + 넓은 콘텐츠. 하단 탭바·FAB 숨김, 대신 상단 우측 `＋ 추가` 버튼(`#hdr-add`). 카드가 **다열 그리드**로: 내역/예산/예측재료 = `column-count:2`, 자산 = 자산/부채 2열 grid. **≥ 1320px = 3열.**
- 히어로·요약·빠른입력은 `max-width`로 과확장 방지(순자산/예산/예측 히어로 640~720px). 카드 좌우 마진은 PC에서 0(그리드 갭이 대신).
- 구현: 같은 HTML/JS, **미디어쿼리로만** 전환(레퍼런스 다이어그램대로). 사이드바 메뉴도 `switchTab()` 호출, 활성표시는 `tb-*`+`sb-*` 동시 토글.
- 검증 필수 폭: **414(폰) / 1024 / 1280 / 1320(3열)**.

## 컨셉
**깔끔한 뉴트럴 베이스(화이트/밝은 그레이) + 프레시 민트 포인트 + 카테고리별 컬러.** 정돈된 느낌은 유지하되 산뜻·재밌게.
색 이력(쓰지 말 것): **블루 거부**, **따뜻한 크림/베이지 거부**, **잉크블랙 단색은 밋밋**. → 무채 베이스 + **민트/그린(#13B68B) 포인트**가 정답.
**이모지 사용 금지.** 얇은 라인(SVG) 아이콘. **폰트 = Apple SD Gothic Neo(시스템 우선)** — 스택 `-apple-system,BlinkMacSystemFont,"Apple SD Gothic Neo",system-ui,"Pretendard Variable",…`. 사용자 아이폰에서 네이티브로 또렷. (Pretendard는 비-애플 폴백)

### 버튼·요소 사이즈 (레퍼런스 비율로 슬림하게 · 2026-06-17 조정)
FAB 48px·폭 min(440,−40) / 시트버튼 48 / 금액입력 52 / 일반입력 46 / 카테고리칩 58 / 세그 39 / 빠른입력·qbtn 42 / hdr-add 38 / 사이드바 44. "너무 크지 않게 적당히"가 기준.

### 소비 다이어리 (적용됨 · 다이어리 중심 큐트)
- **컨셉**: "매일 쓰는 소비 일기장". 게임 메카닉(캐릭터·코인) 대신 **기분+한 줄 일기** 중심.
- **기분 5단계**: `MOODS`(최고/좋음/그냥/별로/후회, 파스텔색). **유니코드 이모지 금지 → 직접 그린 SVG 얼굴**(`moodFace(k,size)`: 색 원 + 점 눈 + `MOUTH[k]` 입모양). 이모지 규칙 유지하며 귀엽게.
- **오늘의 한 줄 카드**(가계부 상단, 당월에만 표시): 파스텔 그라데이션 카드 + 기분 선택 + 한 줄 메모. `DIARY[date]={mood,memo}`, `renderDiaryToday/setMood/saveMemo`.
- **기분 캘린더**(리포트): 무지출 캘린더 셀에 기분 얼굴 오버레이, 탭하면 그날 메모 토스트(`dayTap`).
- **월말 돌아보기**(`recapHTML`): 기분 분포 막대 + 평균기분 한마디.
- 파스텔 톤: 다이어리 카드 mint→lavender 그라데이션, 기분 파스텔 팔레트. (베이스 무채+민트는 유지)

### 재미 요소 (적용됨)
- **카테고리별 컬러**: `CAT_COLOR` 맵(식비=주황 #F97316, 카페=캐러멜, 교통=블루, 생활=틸, 쇼핑=핑크, 의료=레드, 문화=바이올렛, 주거=인디고, 통신=시안, 교육=그린…). 아이콘칩 = `색+'22'`(연한 틴트) 배경 + 풀컬러 아이콘. 리포트 막대도 카테고리색. `catStyle(c)`/`catColor(c)` 헬퍼.
- **숫자 카운트업**: 히어로 금액은 `<span data-cnt="값">0</span>` + `animCounts()` (rAF, 650ms ease-out)로 0→값 또르륵.
- **말랑 인터랙션**: 탭/카드/칩 `:active{transform:scale(.9~.98)}` + cubic-bezier 트랜지션.
- 다크 시그니처 카드(자산 순자산)는 차콜 유지 = 무채 앵커.

## 형태
- 카드: radius `16px`, 그림자 `0 1px 3px rgba(0,0,0,.04)`, **외곽선 없음**
- 리스트: 한 카드 안에 행을 넣고 행 사이는 full-width hairline (애플/토스 그룹리스트)
- 아이콘칩: 36px 원형, 옅은 회색 배경 + 라인 아이콘
- 큰 숫자: 굵게(800), letter-spacing 약간 좁게, tabular-nums
- 시그니처: 자산 순자산은 **다크 그래파이트 카드(#1C1C1E)** 1장으로 포인트

## 아이콘
- 24x24 viewBox, `stroke-width 1.7`, `fill none`, `currentColor`, round cap/join
- 코드 안 `ICONS` 맵 + `icon(name)` 헬퍼로 관리. 새 카테고리 추가 시 여기에 path 추가
- 매핑: 가계부=book, 예산=target, 자산=wallet, 예측=chart / 식비=utensils, 카페=coffee, 교통=bus, 쇼핑=bag, 주거=home, 통신=phone … (코드 ICONS 참고)

## 타이포
- 시스템 폰트 스택(-apple-system / Apple SD Gothic Neo / Pretendard)
- 헤드라인 800, 본문 400~600, 라벨 12px 흐린색

## 톤
- 여백 넉넉히, 색은 적게. 의미(수입/지출/경고)에만 색을 쓰고 나머지는 무채색.
